# ADR - Выбор языка программирования для серверной части

Участники: Команда разработки  
Дата: 22.09.2022
Статус: Проект  

## Контекст

Для серверной части приложения необходимо выбрать язык программирования, который обеспечит высокую производительность, быстрое время разработки и простоту поддержки. Команда имеет опыт работы с Python и JavaScript.

## Рассматриваемые варианты

1. Python
   - Плюсы: Быстрая разработка, обширная экосистема библиотек, легкость в освоении.
   - Минусы: Меньшая производительность по сравнению с компилируемыми языками.
   - Вывод: Подходит для быстрой разработки, но может потребовать оптимизации.

2. JavaScript (Node.js)
   - Плюсы: Высокая скорость разработки, возможность использования одного языка как на клиенте, так и на сервере, хорошая производительность.
   - Минусы: Возможны проблемы с масштабируемостью при больших нагрузках.
   - Вывод: Подходит для приложений с высокой нагрузкой и требует меньше времени на обучение.

3. Java
   - Плюсы: Высокая производительность, хорошая поддержка многопоточности, возможность использования на различных платформах.
   - Минусы: Длительное время разработки, необходимость в более сложной архитектуре.
   - Вывод: Подходит для крупных систем, но может быть избыточным для старта.

## Решение

Выбран Python как основной язык программирования для серверной части.

## Обоснование

Python обеспечивает быструю разработку и позволяет команде сосредоточиться на бизнес-логике, что особенно важно на этапе MVP. Обширная экосистема библиотек также упростит интеграцию с различными API и инструментами.

## Последствия

Выбор Python позволит быстро создать MVP, но в дальнейшем может потребоваться оптимизация производительности. Это решение также облегчит процесс внесения изменений и добавления новых функций.

### Риски

- Низкая производительность под высокой нагрузкой.
- Ограниченная поддержка многопоточности.

## Решение рисков

Для управления производительностью будут использоваться асинхронные библиотеки и подходы, такие как FastAPI или Flask с дополнением к asyncio. Это поможет обеспечить необходимую производительность и устойчивость под нагрузкой.

## Затронутые области

- Серверная часть.
- Архитектура API.
